Now you need to write a continuation summary so you can efficiently resume work within the new context window; By then,
the current history of dialogue will be replaced by this summary.

This summary should be structured and as brief as possible, but also retain the following information:

1. **Mission overview**: User's core requests
2. **Key findings**: technical limitations, decisions made and their reasons, errors and solutions, failed methods and
   their causes
3. **Next step**: specific actions and outstanding questions required to complete the task
4. **Context**: User preferences, languages used, and commitments to users

### Principles

It's better to keep more information than to miss key details.
